Bug Report: Weight Integration Tests Failing
=============================================

Where: weight/tests/integration/conftest.py

What's happening:
4 integration tests in test_get_weight.py are failing. The tests POST a
transaction then GET it back, but the GET returns empty results even though
the POST succeeds (returns 200).

Root cause:
The "client" fixture yields app.test_client() without wrapping it in an app
context. The "setup_and_cleanup" fixture creates its own separate app context.
This means each request (POST, GET) runs in a different app context with a
different database session. Data committed by POST isn't visible to the
subsequent GET.

Fix:
Wrap the test client in a shared app context so all requests within a test
share the same session:

    @pytest.fixture
    def client():
        app.config["TESTING"] = True
        with app.app_context():
            yield app.test_client()

Failing tests:
- test_get_weight_includes_today
- test_response_has_required_fields
- test_containers_is_array
- test_neto_calculated_for_out
