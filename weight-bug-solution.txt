Bug Solution: Why the Fix Works
================================

BACKGROUND CONCEPTS:

  Session:
  When Python wants to talk to the database, it opens a session.
  Think of it as a phone call to the database. While the call is
  open, you can ask questions and send data. The database keeps
  track of what you've said during this call.

  App Context:
  Flask needs to know "which app am I right now?" before it can
  do anything — connect to the database, handle a request, whatever.
  When you open an app context, Flask opens a session (a phone call)
  to the database. When you close the app context, Flask hangs up.

WHAT WAS HAPPENING BEFORE (THE BUG):

  The conftest had two separate fixtures. Each one opened its own
  app context. That means each one made its own phone call to the
  database.

  Phone Call 1 (setup_and_cleanup fixture):
  - "Hey database, delete old test data"
  - "Add test containers TEST-C1 and TEST-C2"
  - "OK bye" — hangs up

  Phone Call 2 (client fixture — where the actual test runs):
  - Test says: POST a truck weighing 15000kg
  - Route handler picks up the phone, tells the database
    "store this transaction", database says OK
  - Test says: GET all transactions from today
  - Route handler picks up the phone, asks the database
    "what transactions do you have?"
  - Database says "let me check my snapshot..." — and the
    snapshot might not include what was just stored, because
    the database is being careful about showing data across
    different calls

  Phone Call 3 (setup_and_cleanup teardown):
  - "Delete test data"
  - "Bye" — hangs up

  Three phone calls for one test. The GET could not see the POST's
  data because they were on different calls with different snapshots.

WHAT WE FIXED:

  One phone call. Everything happens on the same call:

  - "Hey database, delete old test data"
  - "Add test containers"
  - Test POST: "store this transaction" — done
  - Test GET: "show me all transactions" — sees everything
    because it's the same call, same session, same conversation
  - "Delete test data"
  - "OK bye" — hangs up

  Same phone call = same session = everything sees everything.

THE CODE CHANGE:

  Before (two fixtures, two app contexts):

    @pytest.fixture
    def client():
        app.config["TESTING"] = True
        with app.app_context():           # Phone Call 2
            yield app.test_client()

    @pytest.fixture(autouse=True)
    def setup_and_cleanup():
        with app.app_context():           # Phone Call 1
            # setup...
            db.session.commit()
        yield
        with app.app_context():           # Phone Call 3
            # cleanup...
            db.session.commit()

  After (one fixture, one app context):

    @pytest.fixture(autouse=True)
    def client():
        app.config["TESTING"] = True
        with app.app_context():           # One phone call
            # setup...
            db.session.commit()
            yield app.test_client()       # test runs here
            # cleanup...
            db.session.commit()

  Result: 48/48 tests pass.
